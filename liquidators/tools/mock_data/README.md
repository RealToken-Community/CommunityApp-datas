# Mock data

Reference data: CSV exports from the Gitbook documentation (RealT).

Files used to check the consistency of the JSON generated by the tool:

- **Liquidations RMM - Liquidators v2.csv** / **v3.csv**: ranking of liquidators by number of liquidations (used as a reference for totals and top liquidators).
- **Liquidations RMM - RMM v2.csv** / **RMM v3.csv** (and WrapperV3): event details if needed.

To compare the JSON with these mock data files (totals + ranking):

```bash
node compare-mock-data.js
```

The JSON produced by `npm run fetch` (or `node fetch-rmm-liquidators.js`) now contains `v2_total_events`, `v3_total_events`, `liquidators_v2_total`, and `liquidators_v3_total` (with `count` and `amount`), allowing comparison of totals and rankings.

### Build a JSON from mock data only

To recompute a full JSON in the same format as `realt_liquidators_gnosis.json` using **only** the CSV files in this folder (no on-chain call):

```bash
cd .. && node build-json-from-mock-data.js
```

Output: `../data/realt_liquidators_gnosis_from_mock.json` with:

- `v2_total_events` / `v3_total_events`
- `liquidators_v2_total` / `liquidators_v3_total` (amount, count, rank)
- `realt_liquidators_gnosis` by year (2022–2026) with amount, amount_prorata, rank per liquidator

V3 wrapper rows (liquidator = wrapper address) are matched with **Liquidations RMM - WrapperV3.csv** so the real liquidator is used (same order after sorting by date).
